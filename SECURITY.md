# セキュリティポリシー

## サポート対象バージョン

現在、`main`ブランチの最新版のみをサポートしています（MVP段階のためバージョン別のサポートは行っていません）。

## 脆弱性の報告方法

Penguin Control Centerは、システムコマンドの実行・特権操作(`pkexec`)・設定ファイルの書き換え・
外部からのプラグインダウンロードなど、セキュリティ上重要な処理を含みます。脆弱性を発見した場合は、
**公開のIssueには書かず**、以下のいずれかの方法で報告してください。

1. このリポジトリの [Security] タブ → [Report a vulnerability] からGitHub Security Advisoryを作成
2. 上記が使えない場合は、詳細を伏せた上でIssueを作成し「詳細は非公開で連絡したい」旨を記載

### 報告に含めていただきたい情報

- 影響を受けるファイル・機能（例: `core/storeClient.js`のzip展開処理 など）
- 再現手順
- 想定される影響範囲（情報漏洩・任意コード実行・パストラバーサル等）
- 可能であれば修正案

## 設計上のセキュリティ方針（参考）

- コマンド実行は常に配列引数で`execFile`を使用し、シェル文字列の組み立てを行いません
- 特権操作は`pkexec`経由で実行し、実行前に必ず確認ダイアログを表示します
- ストアからのプラグイン導入時は、ダウンロード元のオリジン検証とSHA256によるファイル検証を行います
- `handler.js`を含むプラグインはメインプロセスで無制限に実行されるため、プラグインストアへの掲載は
  提案Issueを経由した公式審査制としています（詳細は[CONTRIBUTING.md](./CONTRIBUTING.md)参照）

## 既知の受容リスク

- プラグインの`handler.js`は制限なしのNode.jsコードとして実行されます。これはプラグインシステムの
  設計上避けられないトレードオフであり、公式審査モデルによってリスクを低減しています。
