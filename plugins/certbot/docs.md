# Certbot (Let's Encrypt証明書)

## 概要

CertbotはLet's Encryptから無料のSSL/TLS証明書を取得・自動更新するための公式ツールです。
本プラグインは証明書の状態確認・更新テストをGUIから行えるようにします。
**新規証明書の発行（ドメイン名・メールアドレスの入力が必要な対話的操作）は対象外**です。
初回発行はターミナルから`certbot --nginx`または`certbot --apache`等を実行してください。

## なぜ新規発行に対応していないか

証明書の新規発行はドメイン名・メールアドレス・利用規約への同意など対話的な入力が必要です。
PCCは「宣言的なCLI定義をボタン化する」設計のため、対話的なウィザード形式のコマンドは
現状GUI化していません。発行済みの証明書の管理（確認・更新テスト・強制更新）に絞ることで、
安全にGUI化できる範囲にとどめています。

## 主なCLIコマンド

| コマンド | 説明 |
|---|---|
| `certbot certificates` | 発行済み証明書の一覧とそれぞれの有効期限を表示 |
| `certbot renew --dry-run` | 実際には更新せず、更新処理が正常に完了するかだけをテスト |
| `certbot renew --force-renewal` | 有効期限に関わらず強制的に証明書を更新 |
| `systemctl status certbot.timer` | 自動更新用のsystemdタイマーの状態を確認 |
| `certbot --version` | インストールされているCertbotのバージョン確認 |

## 自動更新の仕組み

Certbotは通常、`certbot.timer`というsystemdタイマー（`cron`プラグインで扱うcronの代わり）
によって1日2回程度、有効期限が近い証明書を自動的に更新します。証明書の有効期間は90日ですが、
Let's Encryptは残り30日を切った時点からの更新を推奨しており、`certbot renew`はその条件を
満たさない証明書は自動的にスキップします。

## Nginx/Apacheとの連携

証明書を発行・更新すると、Nginx/Apacheプラグインで管理しているWebサーバーの設定にも
証明書のパスが自動的に追記される場合があります（`--nginx`/`--apache`プラグインモードを
使って発行した場合）。更新後にWebサーバーの再起動やreloadが必要になることがあるので、
状況に応じてNginx/Apacheプラグイン側の「設定を再読込」も実行してください。

## よくあるトラブル

- **`certbot`コマンドが権限エラー**: 証明書ファイルはroot権限でのみ読み書きできるため、
  本プラグインの全アクションは`privileged: true`（`pkexec`経由）にしています
- **`renew --dry-run`が失敗する**: Webサーバー側でHTTP-01チャレンジ用のパス
  （`/.well-known/acme-challenge/`）へのアクセスがブロックされていないか確認
- **証明書の有効期限が近いのに自動更新されない**: `certbot.timer`が有効化されているか
  「自動更新タイマーの状態」で確認
