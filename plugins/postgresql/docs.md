# PostgreSQL

## 概要

PostgreSQLは高機能なオープンソースのリレーショナルデータベースです。本プラグインは
MySQL/MariaDBプラグインと同じ設計方針で、**サービスの起動停止のみ**を管理対象とし、
データベース内部の操作（クエリ実行、ユーザー管理、テーブル操作など）は意図的に対象外にしています。

## なぜDB内部の操作に対応していないか

データベースへの接続には認証情報が必要です。PCCはCLIコマンドをそのまま実行する設計上、
認証情報をアプリ内に保存・入力させる機能を持たせたくありません（保存すれば漏洩リスク、
都度入力させれば対話的プロンプトでコマンドが固まるリスクがあります）。そのため本プラグインは
「サービスが動いているかどうか」の管理に留めています。

## 主なCLIコマンド

| コマンド | 説明 |
|---|---|
| `psql --version` | インストールされているクライアントのバージョン確認（認証不要） |
| `systemctl status postgresql.service` | サービスの稼働状態を表示 |
| `pg_isready` | 認証情報なしで接続受付状態のみを確認（DBの中身には触れない） |
| `systemctl restart postgresql.service` | `postgresql.conf`変更後の反映に必要 |
| `pg_lsclusters` | インストールされている全クラスタのバージョン・ポート・状態一覧(Debian/Ubuntu系) |

## 設定ファイル

- パス: `/etc/postgresql/16/main/postgresql.conf`（バージョン番号はインストールされている
  PostgreSQLのメジャーバージョンによって変わります。例: `/etc/postgresql/15/main/`）
- 接続許可設定（どのホスト・ユーザーからの接続を許可するか）は同じディレクトリの
  `pg_hba.conf`で別途管理されています（本プラグインでは未対応）

## バージョン番号について

Debian/Ubuntu系ではPostgreSQLは複数バージョンを共存インストールできる設計になっており、
設定ファイルのパスにメジャーバージョン番号が含まれます。本プラグインは`16`を前提として
いますが、お使いの環境に合わせて`plugin.json`の`configPath`を調整してください。

## データベースの操作をしたい場合

`psql -U <ユーザー名> -d <データベース名>`でターミナルから直接接続してください。
GUI経由でのDB管理ツールが必要な場合は、pgAdminやDBeaverなど専用ツールの利用を推奨します。
