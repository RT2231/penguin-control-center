# UFW (Uncomplicated Firewall)

## 概要

UFWはLinuxのファイアウォール(iptables/nftables)を簡単なコマンドで操作できるフロントエンドです。
本プラグインは状態確認・有効化・ルール確認をGUIから行えるようにします。

## ⚠️ 最重要の注意事項（締め出し防止）

**リモート（SSH）でこのマシンを操作している場合、SSHを許可するルールを追加してから
ファイアウォールを有効化してください。** 許可ルールがない状態で有効化すると、SSH接続が
即座に切断され、二度とリモートから入れなくなる可能性があります。

推奨手順:
1. 「SSHを許可(締め出し防止)」を先に実行
2. 「ファイアウォール状態(詳細)」で許可ルールが反映されたことを確認
3. その後で「ファイアウォールを有効化」を実行

## 主なCLIコマンド

| コマンド | 説明 |
|---|---|
| `ufw status verbose` | 有効/無効の状態と、設定されているルールの詳細を表示 |
| `ufw status numbered` | ルールを番号付きで一覧表示（削除時に番号を指定する） |
| `ufw allow OpenSSH` | SSH(通常22番ポート)への接続を許可 |
| `ufw --force enable` | 確認プロンプトなしで有効化（本アプリのGUI側で既に確認ダイアログを出しているため） |
| `ufw app list` | ufwが認識しているアプリケーションプロファイル一覧（`allow OpenSSH`等で使う名前の一覧） |

## 設定ファイル

- パス: `/etc/default/ufw`
- 個別のルールはこのファイルではなく`ufw`コマンド経由で`/etc/ufw/`配下に反映されます
- このファイルには`IPV6`の有効/無効などデフォルト挙動の設定が含まれます

## よくあるトラブル

- **`ufw`コマンドが権限エラー**: ほぼ全てのufwサブコマンドはroot権限が必要です（本プラグインでは
  状態確認も含め`privileged: true`にしています）
- **SSH接続が切れた**: コンソール（物理/仮想コンソール、クラウドの場合はプロバイダのコンソール機能）
  から`ufw disable`を実行して復旧してください
