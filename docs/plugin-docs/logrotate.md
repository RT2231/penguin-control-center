# logrotate (ログローテーション)

## 概要

logrotateは、肥大化し続けるログファイルを定期的に圧縮・世代管理し、ディスクを圧迫させない
ようにするための定番ツールです。多くのディストリビューションで、`cron`または本プラグインのように
systemdタイマー（`logrotate.timer`）によって1日1回自動実行されます。

## 主なCLIコマンド

| コマンド | 説明 |
|---|---|
| `logrotate -d /etc/logrotate.conf` | dry-run（実際にはローテーションせず、何が実行されるかだけ表示） |
| `logrotate -f /etc/logrotate.conf` | 設定された条件（サイズ・日数）を無視して強制的に今すぐローテーション |
| `ls -la /etc/logrotate.d` | 個別のソフトウェアごとに用意された設定ファイル一覧 |
| `cat /var/lib/logrotate/status` | 各ログファイルが最後にローテーションされた日時の記録 |

## 設定ファイルの書き方（学習用）

`/etc/logrotate.conf`が全体のデフォルト設定、`/etc/logrotate.d/`配下の各ファイルが
ソフトウェアごとの個別設定です。基本構文の例:

```
/var/log/myapp/*.log {
    daily              # 毎日ローテーション(weekly, monthlyも可)
    rotate 7           # 7世代分だけ保持し、それより古いものは削除
    compress           # 古いログをgzip圧縮
    missingok          # ログファイルが存在しなくてもエラーにしない
    notifempty         # 空のログファイルはローテーションしない
    create 0640 www-data adm   # ローテーション後、新しい空ログファイルをこの権限で作成
}
```

## 動作確認の流れ（安全な手順）

1. 「設定ファイルの文法チェック(dry-run)」で、意図した通りの対象ファイルが処理されるか確認
2. 問題なければ「強制的に今すぐ実行」で実際にローテーション

dry-runは実際にファイルを変更しないため、何度実行しても安全です。

## よくあるトラブル

- **ログが全くローテーションされない**: `logrotate.timer`が有効になっているか
  「実行タイマーの状態」で確認
- **`missingok`を付け忘れて、まだ作られていないログでエラーになる**: 該当の個別設定ファイルに
  `missingok`を追加
- **ローテーション後にアプリがログを書き込めなくなった**: アプリケーションによっては
  ログファイルのハンドルを掴んだままなので、`postrotate`セクションでプロセスへの
  シグナル送信（例: `killall -HUP myapp`）が必要な場合があります
