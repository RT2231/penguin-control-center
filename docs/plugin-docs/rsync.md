# rsync (ファイル同期デーモン)

## 概要

rsyncは差分転送によって効率的にファイルを同期するツールです。多くの場合SSH経由の
アドホックなコマンド（`rsync -av src/ user@host:dst/`）として使われますが、
`rsyncd`（デーモンモード）として常駐させることで、バックアップの受け口や
社内向けファイル配布サーバーとしても使えます。本プラグインはこのデーモンモードの
起動停止・状態確認を管理します。

## デーモンモードとSSH経由の違い

| 方式 | 特徴 |
|---|---|
| SSH経由（アドホック） | 暗号化・認証はSSHに委任。追加設定不要で手軽 |
| デーモンモード（本プラグインの対象） | rsync独自のモジュール単位のアクセス制御が可能。ただし平文プロトコル（デフォルト873番ポート）なので、信頼できるネットワーク内での利用や、別途SSHトンネル・VPN越しでの利用を推奨 |

## ⚠️ セキュリティ上の注意

デーモンモードのrsyncはデフォルトで**認証なし・平文通信**です。インターネットに
直接公開すると、設定によっては第三者がファイルを読み書きできてしまいます。
WireGuard/TailscaleプラグインでのVPN経由か、UFWプラグインでの接続元IP制限との
併用を強く推奨します。

## 主なCLIコマンド

| コマンド | 説明 |
|---|---|
| `rsync --version` | インストールされているrsyncのバージョン確認 |
| `systemctl status rsync.service` | デーモンの稼働状態を確認 |

## 設定ファイル

- パス: `/etc/rsyncd.conf`
- `[モジュール名]`セクションで公開するディレクトリを定義します。例:

```ini
[backup]
   path = /srv/backup
   read only = no
   auth users = backupuser
   secrets file = /etc/rsyncd.secrets
```

`auth users`と`secrets file`を設定することで、簡易的なパスワード認証を追加できます
（未設定の場合は誰でもアクセス可能になるため注意）。
