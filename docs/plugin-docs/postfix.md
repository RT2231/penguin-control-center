# Postfix (メールサーバー)

## 概要

PostfixはLinuxで最も広く使われているMTA(メール転送エージェント)です。システムからの
通知メール送信や、独自ドメインでのメール受信サーバーとして使われます。本プラグインは
サービスの状態確認・メールキューの確認・設定チェックをGUIから行えるようにします。

## ⚠️ オープンリレー化に関する重要な注意

Postfixの設定を誤ると、第三者が自由に経由してスパムメールを送信できる
「オープンリレー」状態になる可能性があります。特に`mynetworks`や
`smtpd_relay_restrictions`の設定は、公開サーバーで運用する前に必ず理解してから
変更してください。オープンリレー化すると、メールプロバイダからブロックリスト登録される等
深刻な影響があります。

## 主なCLIコマンド

| コマンド | 説明 |
|---|---|
| `postfix check` | 設定ファイルの文法・パーミッションの整合性をチェック |
| `mailq` | 現在配送待ち・保留中のメールキューを一覧表示 |
| `postfix flush` | キューにあるメールの配送を即座に再試行 |
| `systemctl reload postfix.service` | 設定変更後、接続を切らずに反映 |
| `postconf -n` | デフォルト値から変更されている設定項目のみを一覧表示 |

## 設定ファイル

- パス: `/etc/postfix/main.cf`
- 代表的な項目: `myhostname`（このサーバーのホスト名）、`mynetworks`（リレーを許可する
  送信元ネットワーク）、`relayhost`（外部SMTPリレーを使う場合の転送先）

## メールキューが溜まり続ける場合

`mailq`で配送待ちが大量に溜まっている場合、宛先メールサーバーからの応答が悪い
（スパム判定されている等）か、DNSの逆引き設定（PTRレコード）が不適切な可能性があります。
自宅回線などIPの評判が低い環境からの直接送信は、大手プロバイダ宛てに届きにくいことが
多いため、`relayhost`で信頼できる外部SMTPサービスを経由させる構成もよく使われます。
