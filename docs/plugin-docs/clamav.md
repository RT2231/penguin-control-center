# ClamAV (アンチウイルス)

## 概要

ClamAVはLinux/Unix向けのオープンソースアンチウイルスエンジンです。メールサーバーの
添付ファイルスキャンや、共有ファイルサーバー上のファイルチェックなどに使われます。
本プラグインはデーモン（`clamd`）の状態確認と、ウイルス定義ファイルの更新管理を
GUIから行えるようにします。

## なぜ実際のスキャン実行(clamscan)に対応していないか

`clamscan`によるファイル・ディレクトリのスキャンは、対象パスの指定が必要な上、
大きなディレクトリでは非常に時間がかかり（数分〜数時間）、進捗表示も本アプリの
「ボタンを押して結果を待つ」という設計とは相性がよくありません。将来的に、
進捗表示に対応した専用UIとして追加する可能性はありますが、現時点では対象外です。
実際にスキャンしたい場合はターミナルから`clamscan -r --bell -i /path/to/scan`等を
実行してください。

## 主なCLIコマンド

| コマンド | 説明 |
|---|---|
| `clamscan --version` | ClamAVのバージョンとウイルス定義ファイルの日付を表示 |
| `freshclam` | ウイルス定義ファイルを手動で最新版に更新 |
| `systemctl status clamav-freshclam.service` | 定義ファイル自動更新サービスの状態 |
| `systemctl enable --now clamav-freshclam.service` | 定義ファイルの自動更新を有効化 |
| `systemctl disable --now clamav-freshclam.service` | 定義ファイルの自動更新を無効化 |

## 2つのサービスについて

ClamAVは通常2つのsystemdサービスで構成されます。

| サービス | 役割 |
|---|---|
| `clamav-daemon.service` | 常駐してスキャン要求を処理する本体(`clamd`) |
| `clamav-freshclam.service` | ウイルス定義ファイルを定期的に自動更新するデーモン |

## 設定ファイル

- パス: `/etc/clamav/clamd.conf`
- 代表的な項目: `MaxFileSize`（スキャン対象の最大ファイルサイズ）、
  `LocalSocket`（他プロセスからclamdに接続するためのソケットパス）

## よくあるトラブル

- **`freshclam`の手動更新が失敗する**: `clamav-freshclam.service`が既に起動中の場合、
  ロックファイルが競合してエラーになることがあります。まずサービスを停止してから
  手動更新を試してください
- **メモリ使用量が多い**: ウイルス定義データベース全体をメモリに読み込むため、
  他のプロセスと比べてメモリ使用量が大きくなる傾向があります
