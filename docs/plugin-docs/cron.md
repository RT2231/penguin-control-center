# cron (定期実行タスク)

## 概要

cronは「毎日3時にバックアップを実行する」のように、指定した時刻・間隔でコマンドを自動実行する
ためのデーモンです。本プラグインはサービスの状態確認と、登録済みタスクの一覧確認をGUIから行えるように
します（タスクの追加・編集は現時点では非対応。下記のcrontab書式を参考に`crontab -e`で行ってください）。

## 主なCLIコマンド

| コマンド | 説明 |
|---|---|
| `crontab -l` | 現在のユーザーに登録されているcrontabを一覧表示（未登録の場合は「no crontab」と表示） |
| `crontab -e` | crontabを編集（本プラグイン未対応、ターミナルから直接実行） |
| `ls -la /etc/cron.d` | システム全体（rootや他ユーザー）が登録した定期タスクの設定ファイル一覧 |

## crontab書式チートシート（学習用）

crontabの1行は次の6つのフィールドで構成されます（左から順に）。

```
分(0-59) 時(0-23) 日(1-31) 月(1-12) 曜日(0-7,0と7は日曜) コマンド
```

| 書き方 | 意味 |
|---|---|
| `* * * * *` | 毎分実行 |
| `0 3 * * *` | 毎日3時0分に実行 |
| `*/15 * * * *` | 15分ごとに実行 |
| `0 9 * * 1-5` | 平日(月〜金)の9時に実行 |
| `0 0 1 * *` | 毎月1日の0時に実行 |

例: 毎日深夜3時にバックアップスクリプトを実行する場合

```
0 3 * * * /home/user/backup.sh
```

## 設定ファイル・登録場所の種類

| 場所 | 用途 |
|---|---|
| `crontab -e`（ユーザー個別） | 特定ユーザーの権限で実行したいタスク |
| `/etc/crontab` | システム全体の設定（実行ユーザーを指定するフィールドが追加される） |
| `/etc/cron.d/` | パッケージ等が独自に配置する定期タスク定義 |
| `/etc/cron.daily/` 等 | シェルスクリプトを置くだけで毎日/毎週/毎月実行されるディレクトリ |

## 最近の代替: systemdタイマー

現代的なLinuxディストリビューションでは、cronの代わりに`systemd`のタイマー機能（`.timer`ユニット）
を使う設計も増えています。ログ管理や依存関係の指定がしやすい反面、書式はcrontabより複雑です。
本プラグインは伝統的なcronのみを対象としています。

## よくあるトラブル

- **登録したはずのタスクが動かない**: 環境変数(`PATH`等)がログインシェルと異なるため、
  スクリプト内でフルパスを使っていないことが原因の場合が多いです
- **`crontab -l`で「no crontab for user」と出る**: そのユーザーにはまだタスクが登録されていません
  （エラーではなく正常な状態です）
