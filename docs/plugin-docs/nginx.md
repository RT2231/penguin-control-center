# Nginx

## 概要

NginxはApacheと並ぶ代表的なWebサーバー/リバースプロキシです。本プラグインは`nginx`サービスの
起動停止・設定文法チェックをGUIから行えるようにします。

## 主なCLIコマンド

| コマンド | 説明 |
|---|---|
| `nginx -t` | 設定ファイルの文法チェック（`syntax is ok` / `test is successful`が出れば問題なし） |
| `nginx -V` | バージョンとビルド時のコンパイルオプションを表示 |
| `systemctl reload nginx.service` | 接続を切らずに設定を反映（`restart`より安全） |

## 設定ファイル

- パス: `/etc/nginx/nginx.conf`
- サイト個別設定は通常 `/etc/nginx/sites-available/` に置き、`/etc/nginx/sites-enabled/`へ
  シンボリックリンクすることで有効化します（本プラグインでは`nginx.conf`本体のみ編集対象、
  個別サイト設定は将来拡張予定）
- **重要**: 設定変更後は必ず`nginx -t`を実行してから`reload`/`restart`してください

## Apacheとの併用について

NginxとApacheは両方とも80/443番ポートを使おうとするため、同一マシンで両方を起動していると
どちらかが起動失敗します。片方をリバースプロキシ、もう片方をバックエンドにする構成も可能ですが、
その場合はポート番号を分けるなどの追加設定が必要です。

## よくあるトラブル

- **`nginx -t`でエラー**: エラーメッセージにファイル名と行番号が出るので該当箇所を確認
- **起動しない（ポート競合）**: Apache等、他のプロセスが80/443番ポートを使用していないか確認
- **設定ファイルの読み取り権限エラー**: `nginx -t`は通常一般ユーザーでも実行できますが、
  権限が絞られた環境では失敗する場合があります
