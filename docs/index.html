OSS / Linux GUI管理プラットフォーム

🐧 GUIで操作するたびに、
対応するCLIが見える。

Penguin Control Centerは、Linuxの管理系CLIツールをGUIから安全に操作しながら、 その裏で実行されているコマンドをそのまま学べる、プラグイン型のオープンソース管理ツールです。

GitHubで見る コントリビュートする プラグインを公開する
同期状態を確認
chronyc tracking
再起動
systemctl restart chrony.service
参照サーバー一覧
chronyc sources -v
実際のGUI設定タブの表示例（chronyプラグイン）。オレンジのバーは管理者権限が必要な操作。

3つの柱

01

GUI管理ツール

CLIのみで管理されるソフトウェアを、GUIから安全に設定・操作。

02

学習ツール

GUI操作と対応するCLI・設定ファイルを並べて表示し、実務のLinux操作を学べる。

03

プラグインプラットフォーム

JSONマニフェスト中心の宣言的な形式で、誰でも新しいソフトウェアへの対応を追加できる。

本体（コア）にPRを送る

  1. フォーク & セットアップ

    gh repo fork RT2231/penguin-control-center --clone && npm install

  2. ブランチを切る

    feat/ fix/ などのプレフィックスを付けたブランチ名にする。

  3. 構文チェック

    変更したJS/JSONファイルを node --check で確認してからコミット。

  4. PR作成

    gh pr create --base main で変更内容・動作確認方法を明記して提出。

→ 詳しいコーディング規約とレビュー基準はCONTRIBUTING.md

プラグインを公開する

plugins/<id>/ 配下に plugin.json を置くだけで、新しいソフトウェアへの対応をGUIに追加できます。 複雑な処理が必要な場合のみ handler.js を追加してください。

  1. マニフェストを書く

    id・name・actions（CLIは配列）を定義。

  2. ドキュメントを書く

    docs.md に概要・主なCLI・トラブルシューティングを記載。

  3. PRを出す

    plugin/<id> ブランチで提出。セキュリティレビュー基準を満たしているか確認されます。

{
  "id": "openssh",
  "name": "OpenSSH",
  "version": "0.1.0",
  "description": "...",
  "service": {
    "systemdUnit": "ssh.service",
    "configPath": "/etc/ssh/sshd_config"
  },
  "actions": [
    {
      "id": "status",
      "label": "状態確認",
      "cli": ["systemctl", "status",
              "ssh.service", "--no-pager"],
      "privileged": false
    }
  ],
  "docs": "docs.md"
}
